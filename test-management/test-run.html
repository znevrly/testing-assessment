Test Runs

We will conduct several test runs to verify functionality and reduce risks. Each test run focuses on different aspects of the user stories.

Test Run 1: Basic Functionality

Goal

  • Quickly confirm that main features for each user story work as expected (happy paths).

Scope

  • New Device Login: Confirm 2FA flow works once.
  • Upgrade to Premium: Check payment succeeds.
  • Approval Notifications (SMS): One basic approval triggers correct SMS.
  • Departments Approval: Simple multi-department flow.
  • CR Verification: Upload valid doc, see “Verified” status.

Outcome

  • If basic acceptance tests pass, move to next run.

Test Run 2: Negative & Edge Cases

Goal

  • Test unusual situations and invalid inputs.

Scope

  • New Device Login: Wrong 2FA codes, brute-force attempts.
  • Upgrade to Premium: Invalid payment data, payment gateway timeouts.
  • Approval Notifications (SMS): Invalid phone number, gateway downtime.
  • Departments Approval: Missing departments, rejection flows.
  • CR Verification: Expired or wrong file format, external API offline.

Outcome

  • We log defects found here for quick fix, especially if they block main functionality.

Test Run 3: Integration Checks

Goal

  • Confirm app talks correctly to external services (payment, SMS, CR).

Scope

  • Payment integration for premium upgrade (approval codes, refunds if needed).
  • SMS gateway calls for notifications.
  • CR verification with external or mock API.

Outcome

  • Ensure no breakdown between app and third-party services.

Test Run 4: Regression

Goal

  • Make sure new fixes did not break previous functionality.

Scope

  • Rerun critical tests from all user stories:
    • New Device Login, Premium Upgrade, Approvals (SMS, Departments), CR Verification
  • Check previously reported bugs are resolved or still open.

Outcome

  • If no major regressions, we prepare for UAT.

Test Run 5: UAT

Goal

  • Real users or stakeholders validate system meets business needs.

Scope

  • Business scenarios for each user story.
  • Final check on user experience, data correctness.

Outcome

  • UAT sign-off indicates readiness for production.
